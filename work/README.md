# 작업 보드

제안되었거나 진행 중인 기능, 버그 수정, 동작 변경마다 이 디렉터리에 Markdown
파일 하나를 둠.

아직 확정되지 않은 장기 계획과 개발 순서는 `docs/ROADMAP.md`에서 관리함.
곧 논의하거나 실행할 수 있을 정도로 구체화된 항목만 이 디렉터리로 옮김.

## 생명주기

1. `TEMPLATE.md`를 복사하여 `001-initial-imap-connection.md`처럼 다음 순번의
   파일을 생성함.
2. 요구사항을 결정하는 동안 상태를 `논의 중`으로 둠.
3. 완료 조건이 정해지고 미결 사항이 해결되면 상태를 `준비됨`으로 변경함.
4. 구현 중에는 상태를 `진행 중`으로 변경함.
5. 결정, 범위, 검증 결과를 최신 상태로 유지함.
6. 구현, 검증, 스펙 동기화가 끝나면 작업 파일을 삭제함. Git 이력이 보관소임.

이 디렉터리에는 `README.md`, `TEMPLATE.md`, 진행 중인 작업 항목만 존재해야 함.
