# MMCP 개발 로드맵

## 문서 성격

이 문서는 아직 확정되지 않은 장기 계획과 예상 개발 순서를 관리함. 각 단계의
구체적인 요구사항을 논의하거나 구현할 때는 별도의 `work/` 작업 항목을 생성함.
완료되어 현재 요구 동작이 된 내용은 `docs/SPEC.md`에 반영함.

## 예상 개발 순서

### 1단계: 기술 기반 결정

- 네이버 메일과 애플리케이션 비밀번호 기반 IMAP 인증 확인 완료
- Node.js, TypeScript, 공식 MCP SDK, `imapflow`, `mailparser` 선택 완료
- MCP 접속 지점 인증 방식 선택
- Streamable HTTP 기반 최소 MCP 서버 구현 완료

### 2단계: 읽기 전용 이메일 조회

- IMAP 연결과 편지함 목록 조회 구현 완료
- 이메일 검색과 결과 개수 제한 구현 완료
- 이메일 메타데이터 및 안전한 텍스트 본문 조회 구현 완료
- 이메일 내용 정제 및 비밀정보 없는 로그 처리 구현 완료

### 3단계: 수신 이메일 상태 관리

- 읽음 및 읽지 않음 상태 변경 구현 완료
- 편지함 간 이동 구현 완료
- 네이버가 안정적인 `\Archive` 특수 편지함을 제공하지 않음을 확인하고 보관
  기능 제외
- 상태 변경 작업의 입력 검증과 오류 처리 구현 완료

### 4단계: 운영 준비

- HTTPS 및 MCP 접속 인증 구성
- 서비스 자동 시작과 장애 후 재시작 구성
- 비밀값 관리, 로그 순환, 업데이트 절차 구성
- 실제 모바일 ChatGPT 및 Codex 연결 검증

### 5단계: 라즈베리파이 배포 검토

- 안정된 MMCP의 실제 CPU, 메모리, 저장 공간 사용량 측정
- Raspberry Pi 3를 우선 대상 장비로 검증
- 필요하면 Raspberry Pi 2 호환성 검증
- ARM 아키텍처용 배포물과 설치 절차 검증
- 장시간 실행, 네트워크 재연결, 전원 복구 검증
- microSD 손상 위험을 줄이기 위한 로그 및 쓰기 빈도 검토

## 향후 검토 후보

### 이메일 관리 실용 기능 확장 우선순위

1. 검색 UID 페이지네이션과 별표·크기 조건
2. 이동·복사 성공 결과의 목적지 UID 노출
3. 여러 편지함 상태 요약 조회
4. 벌크 작업 장애 복구
5. 제한적인 분류 보조 메타데이터 추출
6. 첨부파일 텍스트 추출

- dry-run과 custom flag는 실제 필요성이 확인될 때 다시 검토
- 로컬 검색 색인
- 범용 IMAP 환경의 thread 재구성
- 여러 메일 계정 지원
- SMTP 및 이메일 발송

위 항목은 현재 스펙에 포함되지 않으며 구현을 약속하지 않음.
