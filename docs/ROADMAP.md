# MMCP 개발 로드맵

## 문서 성격

이 문서는 아직 확정되지 않은 장기 계획과 예상 개발 순서를 관리함. 각 단계의
구체적인 요구사항을 논의하거나 구현할 때는 별도의 `work/` 작업 항목을 생성함.
완료되어 현재 요구 동작이 된 내용은 `docs/SPEC.md`에 반영함.

## 예상 개발 순서

### 1단계: 기술 기반 결정

- 메일 제공자와 IMAP 인증 방식 확인
- 구현 언어와 주요 라이브러리 선택
- MCP 접속 지점 인증 방식 선택
- Streamable HTTP 기반 최소 MCP 서버 실행

### 2단계: 읽기 전용 이메일 조회

- IMAP 연결과 편지함 목록 조회
- 이메일 검색과 페이지 구분
- 이메일 메타데이터 및 안전한 텍스트 본문 조회
- 이메일 내용 정제 및 비밀정보 없는 로그 처리

### 3단계: 수신 이메일 상태 관리

- 읽음 및 읽지 않음 상태 변경
- 편지함 간 이동
- 제공자별 보관 동작 확인 및 구현
- 상태 변경 작업의 입력 검증과 오류 복구

### 4단계: 운영 준비

- HTTPS 및 MCP 접속 인증 구성
- 서비스 자동 시작과 장애 후 재시작 구성
- 비밀값 관리, 로그 순환, 업데이트 절차 구성
- 실제 모바일 ChatGPT 및 Codex 연결 검증

### 5단계: 라즈베리파이 배포 검토

- 안정된 MMCP의 실제 CPU, 메모리, 저장 공간 사용량 측정
- Raspberry Pi 3를 우선 대상 장비로 검증
- 필요하면 Raspberry Pi 2 호환성 검증
- ARM 아키텍처용 배포물과 설치 절차 검증
- 장시간 실행, 네트워크 재연결, 전원 복구 검증
- microSD 손상 위험을 줄이기 위한 로그 및 쓰기 빈도 검토

## 향후 검토 후보

- 첨부파일 다운로드 및 내용 처리
- 로컬 검색 색인
- 범용 IMAP 환경의 thread 재구성
- 여러 메일 계정 지원
- SMTP 및 이메일 발송

위 항목은 현재 스펙에 포함되지 않으며 구현을 약속하지 않음.
