# 이메일 변경 도구를 복수형 벌크 도구로 제공

## 상태

승인됨

## 배경

ChatGPT에서 단일 이메일 변경 도구를 여러 번 호출하면 같은 정리 작업에도
도구 사용 승인이 이메일마다 반복됨. IMAP은 여러 명령을 transaction으로 묶지
않으므로 호출 단위를 합치더라도 전체 작업의 원자성은 보장할 수 없음.

## 결정

- 읽음, 별표, 복사, 이동, 휴지통, 스팸 도구를 복수형 벌크 도구로 제공함.
- 각 호출은 작업별 고유 ID를 가진 최대 100개 작업을 받음.
- 작업별 출발 편지함, 목적지 편지함과 상태를 허용함.
- 입력 구조와 중복 작업은 전체 실행 전에 검증함.
- 초기 구현은 정확한 개별 실패 식별을 위해 작업별로 순차 실행함.
- 실행 중 개별 오류는 실패 목록에 기록하고 다음 작업을 계속함.
- 요약과 함께 모든 작업의 `succeeded` 또는 `failed` 결과를 입력 순서대로
  반환함.
- 장애 복구와 실행 저널은 별도 후속 작업으로 구현함.

## 결과

- 여러 이메일 변경을 한 번의 도구 승인으로 수행할 수 있음.
- 서로 다른 편지함과 목적지 작업을 한 호출에 포함할 수 있음.
- 일부 성공과 응답 유실 가능성은 남으며 transaction과 rollback은 제공하지
  않음.
