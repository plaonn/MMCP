# Streamable HTTP만 사용

## 상태

승인됨

## 배경

MMCP의 주 사용 환경은 모바일 ChatGPT임. MCP 서버는 사용자의 기존 DDNS
환경을 통해 외부에서 접근 가능해야 함. 주 사용 사례에는 로컬 프로세스 연동이
필요하지 않음.

Codex도 Streamable HTTP MCP 서버에 연결할 수 있으므로 하나의 HTTP 서버로
의도한 두 종류의 클라이언트를 모두 지원할 수 있음.

## 결정

MMCP는 MCP Streamable HTTP 전송 방식만 제공함.

향후 요구사항이 이 결정을 변경할 충분한 이유를 제공하지 않는 한 STDIO
전송 방식을 구현하거나 유지하지 않음.

## 결과

- 하나의 전송 방식으로 ChatGPT와 호환되는 Codex 클라이언트를 모두 지원함.
- 외부에서 접근 가능한 HTTPS 접속 지점과 접속 지점 인증이 필요함.
- 로컬 개발 및 테스트에서도 HTTP 전송 방식을 사용해야 함.
- 별도의 로컬 프로세스 진입점을 유지하는 비용이 없음.

## 검토한 대안

- Streamable HTTP와 STDIO를 모두 지원하는 방안은 STDIO가 모바일 중심 사용
  사례에 기여하지 않으며 구현과 테스트 비용을 추가하므로 제외함.
- STDIO만 지원하는 방안은 모바일 ChatGPT가 로컬 STDIO 프로세스를 실행하거나
  연결할 수 없으므로 제외함.
