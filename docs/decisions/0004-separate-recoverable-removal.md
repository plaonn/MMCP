# 복구 가능한 이동과 영구 삭제 분리

## 상태

승인됨

## 배경

IMAP은 휴지통 이동, `\Deleted` flag, `EXPUNGE`, 편지함 삭제 등 여러 삭제 관련
동작을 제공함. MMCP는 모델이 호출하는 개인 메일 관리 서버이므로 복구 가능한
정리 동작은 유용하지만, 잘못된 식별자나 의도 해석으로 영구 손실이 발생하는
경로는 제공하면 안 됨.

일반 편지함 이동에 휴지통과 스팸을 포함하면 호출 의도가 도구 목록과
annotation에 명확히 드러나지 않음.

## 결정

- 영구 삭제, `\Deleted` flag 설정, `EXPUNGE`, 휴지통 비우기를 제공하지 않음.
- 내부 이메일을 영구 손실시킬 수 있는 편지함 삭제를 제공하지 않음.
- 명시적으로 지정된 이메일의 휴지통 이동은 `trash_emails` 전용 도구로
  제공하고
  `destructiveHint`를 표시함.
- 명시적으로 지정된 이메일의 스팸 이동은 `mark_emails_as_spam` 전용 도구로
  제공함.
- 일반 `move_emails`는 `\Trash`와 `\Junk` 특수 편지함 이동을 거부함.

## 결과

- 사용자는 복구 가능한 휴지통 이동과 스팸 처리를 사용할 수 있음.
- 모델과 사용자는 일반 이동과 삭제 의도를 도구 수준에서 구분할 수 있음.
- MMCP를 통해 이메일이나 편지함을 영구 삭제할 수 없음.
