# 자연어 정책에 구조화된 patch 사용

## 상태

승인됨

## 배경

MMCP의 메일 관리 정책은 GPT가 사람의 문맥처럼 해석하는 자연어 사용자 지침임.
정책 전문 전체를 GPT가 다시 작성해 교체하면 기존 규칙 누락, 의도하지 않은
의미 변경, 다른 대화에서 수정한 최신 정책 덮어쓰기 위험이 있음. 전체 전문
변경은 사용자와 플랫폼의 안전 검사에서도 변경 범위를 판단하기 어려움.

## 결정

- 정책은 안정적인 규칙 ID와 자연어 본문 목록으로 저장함.
- 정책 전문 전체 교체 도구를 제공하지 않음.
- 정책 변경은 규칙 단위 `add`, `replace`, `remove` operation만 허용함.
- 변경 전 상태를 바꾸지 않는 미리보기와 구조화된 diff를 제공함.
- 적용과 복원은 `expectedRevision`이 현재 revision과 일치할 때만 수행함.
- 최근 revision 이력을 최대 20개 보관하고, 과거 상태 복원도 새 revision으로
  기록함.
- 정책은 저장소 외부 로컬 JSON 파일에 atomic write함.
- 최신 정책은 MCP server instructions에 포함하고, 상세 판단 전 정책 조회
  도구를 사용하도록 안내함.

## 결과

- GPT가 정책 전체를 재작성하지 않고 의도한 작은 변경만 제안하고 적용할 수
  있음.
- 정책 변경 범위와 이전·이후 값이 명확하게 보임.
- 동시 수정 충돌을 감지하고 과거 정책으로 복구할 수 있음.
- 자연어 정책은 판단 지침이며, 영구 삭제 금지 같은 코드 안전 규칙을 변경하지
  못함.
