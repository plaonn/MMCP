# 벌크 이메일 조회 본문 크기 제한

## 상태

승인됨

## 배경

`get_emails`가 여러 이메일의 전체 안전 텍스트를 한 응답에 포함하면 긴 본문
몇 건만으로도 ChatGPT tool response budget을 초과할 수 있음. 응답 뒤쪽이
잘리면 도구 실행이 성공해도 모델이 일부 작업 결과를 확인하지 못함.

## 결정

- `get_emails`는 기본적으로 이메일별 최대 2,000자의 본문 미리보기를 반환함.
- 호출 전체에서 반환하는 본문은 최대 20,000자로 제한함.
- 남은 전체 본문 예산을 남은 작업 수로 균등 배분하여 입력 순서 뒤쪽 작업도
  본문 예산을 받게 함.
- 호출자는 본문과 첨부파일 메타데이터 포함 여부, 이메일별 본문 최대 길이를
  제한 범위 안에서 지정할 수 있음.
- 각 성공 결과는 정제된 전체 텍스트 길이, 잘림 여부와 잘림 이유를 반환함.
- 문자 길이와 절단은 Unicode code point 기준으로 처리함.
- 벌크 조회에서는 제한 없는 전체 본문을 제공하지 않으며 기존 단건
  `get_email`로 조회함.

## 결과

- 여러 이메일 분류 시 응답 뒤쪽 작업이 잘릴 가능성을 줄임.
- 모델은 본문이 전체인지 미리보기인지 명확하게 구분할 수 있음.
- 전체 본문이 필요한 이메일은 추가 단건 조회가 필요함.
- 본문 이외 메타데이터 크기와 client 자체 응답 제한으로 인한 잘림 가능성은
  남음.

## 검토한 대안

- 벌크 조회에서 전체 본문 요청 허용: 호출자가 응답 제한을 쉽게 우회하여 같은
  문제가 반복될 수 있어 제외함.
- 첫 작업부터 요청 한도만큼 순차 사용: 앞쪽 작업이 전체 예산을 소비하여 뒤쪽
  결과가 다시 잘릴 수 있어 제외함.
- 실제 token 수 기준 제한: client와 모델별 token 계산을 서버가 안정적으로
  예측할 수 없어 문자 수 기준을 사용함.
